203=head1 NAME
204
205Log::Log4perl::Config::Watch - Detect file changes
206
207=head1 SYNOPSIS
208
209 use Log::Log4perl::Config::Watch;
210
211 my $watcher = Log::Log4perl::Config::Watch->new(
212 file => "/data/my.conf",
213 check_interval => 30,
214 );
215
216 while(1) {
217 if($watcher->change_detected()) {
218 print "Change detected!\n";
219 }
220 sleep(1);
221 }
222
223=head1 DESCRIPTION
224
225This module helps detecting changes in files. Although it comes with the
226C<Log::Log4perl> distribution, it can be used independently.
227
228The constructor defines the file to be watched and the check interval
229in seconds. Subsequent calls to C<change_detected()> will
230
231=over 4
232
233=item *
234
235return a false value immediately without doing physical file checks
236if C<check_interval> hasn't elapsed.
237
238=item *
239
240perform a physical test on the specified file if the number
241of seconds specified in C<check_interval>
242have elapsed since the last physical check. If the file's modification
243date has changed since the last physical check, it will return a true
244value, otherwise a false value is returned.
245
246=back
247
248Bottom line: C<check_interval> allows you to call the function
249C<change_detected()> as often as you like, without paying the performing
250a significant performance penalty because file system operations
251are being performed (however, you pay the price of not knowing about
252file changes until C<check_interval> seconds have elapsed).
253
254The module clearly distinguishes system time from file system time.
255If your (e.g. NFS mounted) file system is off by a constant amount
256of time compared to the executing computer's clock, it'll just
257work fine.
258
259To disable the resource-saving delay feature, just set C<check_interval>
260to 0 and C<change_detected()> will run a physical file test on
261every call.
262
263If you already have the current time available, you can pass it
264on to C<change_detected()> as an optional parameter, like in
265
266 change_detected($time)
267
268which then won't trigger a call to C<time()>, but use the value
269provided.
270
271=head2 SIGNAL MODE
272
273Instead of polling time and file changes, C<new()> can be instructed
274to set up a signal handler. If you call the constructor like
275
276 my $watcher = Log::Log4perl::Config::Watch->new(
277 file => "/data/my.conf",
278 signal => 'HUP'
279 );
280
281then a signal handler will be installed, setting the object's variable
282C<$self-E<gt>{signal_caught}> to a true value when the signal arrives.
283Comes with all the problems that signal handlers go along with.
284
285=head2 TRIGGER CHECKS
286
287To trigger a physical file check on the next call to C<change_detected()>
288regardless if C<check_interval> has expired or not, call
289
290 $watcher->force_next_check();
291
292on the watcher object.
293
294=head2 DETECT MOVED FILES
295
296The watcher can also be used to detect files that have moved. It will
297not only detect if a watched file has disappeared, but also if it has
298been replaced by a new file in the meantime.
299
300 my $watcher = Log::Log4perl::Config::Watch->new(
301 file => "/data/my.conf",
302 check_interval => 30,
303 );
304
305 while(1) {
306 if($watcher->file_has_moved()) {
307 print "File has moved!\n";
308 }
309 sleep(1);
310 }
311
312The parameters C<check_interval> and C<signal> limit the number of physical
313file system checks, simililarily as with C<change_detected()>.
